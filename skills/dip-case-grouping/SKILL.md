# DIP分组核算技能

使用本技能时，不要把任务理解成单纯“病历找病种编码”。默认目标是完成一条完整的 DIP 核算链路：

1. 检查数据是否齐备
2. 将国临版诊断/手术编码转换为医保版编码
3. 按主诊断 + 治疗方式完成病例入组
4. 计算或读取病种分值、点值、预计结算金额与结余
5. 输出病例、科室、病组三个层级的核算结果
6. 对未入组、部分命中、亏损病例做原因分析

## 先读哪些文件

- 规则依据：先读 [references/dip-rules.md](./references/dip-rules.md)
- 本项目数据映射：再读 [references/project-dip-data-map.md](./references/project-dip-data-map.md)
- 执行前检查：需要时运行 [scripts/inspect_dip_inputs.py](./scripts/inspect_dip_inputs.py)

如果用户只是问“DIP规则是什么”，只需读规则依据。
如果用户要在本项目里算结果、查问题、生成结论，两个 reference 都要读。

## 标准工作流

### 1. 先确认任务属于哪一类

- `规则解释类`：解释核心病种/综合病种、点值、支付标准、辅助目录、异常调校
- `入组核算类`：按月份或病例范围重算 DIP，输出 match rate、score、estimated settlement、balance
- `问题排查类`：分析未入组、编码未映射、主手术缺失、目录库未命中、亏损病例
- `经营分析类`：按科室、病组、月份输出对比和管理建议

### 2. 做数据准备检查

优先确认以下对象是否可用：

- `数据仓库/平顶山2025年DIP2.0分组目录库.xlsx`
- `数据仓库/ICD10国临版2.0对照医保版2.0.xlsx`
- `数据仓库/ICD9国临版3.0对照医保版2.0.xlsx`
- `dwd_case`
- `dws_case_fee_summary`
- `sys_dip_point_value`

可先运行：

```powershell
python skills/dip-case-grouping/scripts/inspect_dip_inputs.py
```

如果用户明确只分析某个月，可带月份：

```powershell
python skills/dip-case-grouping/scripts/inspect_dip_inputs.py --stat-month 2025-01
```

### 3. 执行核算时遵循的口径

- 入组前必须先用 ICD 对照表把诊断编码、手术编码转换成医保编码
- 诊断使用 `ICD10国临版2.0对照医保版2.0.xlsx`
- 手术使用 `ICD9国临版3.0对照医保版2.0.xlsx`
- 如果编码未对照成医保码，不应直接拿原始临床码参与正式入组
- 对 `6.31 -> 06.3100`、`4.43 -> 04.4300` 这类前导零差异，允许在 ICD9 对照表内做归一化后再入组
- 诊断编码与手术编码优先统一为医保版编码后再入组
- 核心病种按 `医保版主诊断前 5 位（保留小数点）+ 核心病组操作编码` 精确匹配
- 综合病种按 `医保版主诊断前三位 + 治疗方式属性` 匹配
- 治疗方式属性固定为 4 类：
  `BSZL` 保守治疗、`ZDXCZ` 诊断性操作、`ZLXCZ` 治疗性操作、`SS` 手术
- 当前项目口径下：
  任何病例只要存在“诊断前 5 位 + 操作编码”的核心精确行，都先进入核心病组；
  `SS` 在没有精确核心行时，回退到综合手术组；
  `ZDXCZ`、`ZLXCZ` 在没有精确核心行时，回退到对应综合病组；
  无操作编码病例先尝试按诊断前 5 位命中核心病种；
  若核心病种无对应诊断直匹配，再进入 `BSZL`
- 多候选命中时，优先核心病种，再优先分值更高者
- 点值为空时，可以先输出“部分就绪”的入组与分值结果，但不能伪造结算金额
- 输出经营分析时，明确区分 `matched` 与 `unmatched`；旧逻辑中的 `partial` 仅用于历史结果解释，不应再作为新的综合病种正式入组口径
#### 3.1 特殊病例入组口径
- 多诊断病例：主诊断未命中时，依次使用第一、第二副诊断尝试入组，标记为「副诊断入组」
- 多手术病例：主手术未命中时，按手术级别从高到低依次尝试匹配，联合手术优先匹配专属病组
- 新生儿病例（年龄<28天）：优先匹配新生儿专属病种，按出生体重分层入组
- 重症病例（ICU>7天、有创呼吸机>96小时、CRRT）：优先匹配重症专属病种，自动触发CCI校正
- 肿瘤病例：区分手术/放化疗/靶向治疗等不同治疗方式匹配对应病种，转移癌触发校正
- 日间手术/康复病例：匹配对应专属目录，适用特殊结算规则
- 先期分组病例（器官移植、造血干细胞移植、重度烧伤等）：直接进入先期分组，优先级最高

## 本项目中的推荐执行方式

### 场景1：用户要“重算 DIP”

优先使用现有服务逻辑或对应 API，目标是刷新以下结果表：

- `dwd_case_dip_match`
- `dws_dip_case_result`
- `ads_dip_monthly_hospital_metrics`
- `ads_dip_monthly_dept_metrics`
- `ads_dip_monthly_group_metrics`

需要说明：

- 若指定 `stat_month` 且已有激活点值，可输出完整结算结果
- 若未指定月份或无点值，只能输出分组/分值层结果，`calc_mode` 应保持 `partial_ready`

### 场景2：用户要“分析为什么没入组”

优先检查：

- 主诊断是否已映射到医保版编码
- 主手术是否已被识别为 `SS` / `ZDXCZ` / `ZLXCZ`
- 手术病例是否未命中核心病组，且综合 `SS` 也未命中
- 诊断性/治疗性操作是否未命中对应综合病组
- 无操作病例是否未命中 `BSZL`

输出时按原因分布、典型病例、可修复项三个维度整理。

### 场景3：用户要“看科室亏损/结余”

先确认该月份存在激活点值，再按以下逻辑分析：

- `estimated_settlement_amount = score_value * point_value`
- `balance_amount = estimated_settlement_amount - total_fee`
- `profit_flag` 分为 `surplus`、`loss`、`balanced`、`unknown`

默认同时看：

- 科室总分值
- 预计结算金额
- 实际总费用
- 总结余/亏损
- 亏损病例数与占比
- 主要亏损病组

### 场景4：用户要“分析特殊病例入组”
针对新生儿、肿瘤、重症、多手术、多诊断等特殊病例：
- 优先检查是否匹配到对应专属病种
- 检查辅助目录校正系数是否正确应用
- 分析未入组/分值偏低的原因，给出优化建议
- 输出特殊病例入组率、校正后分值变化情况

### 场景5：用户要“分析异常病例”
针对超高/低费用、超长住院、二次入院等异常病例：
- 识别符合特病单议条件的病例，标记并给出申报建议
- 计算异常病例的校正后结算金额和盈亏情况
- 分析异常病例的分布规律和主要原因
- 给出异常病例管理的优化建议

### 场景6：用户要“核算特病单议病例”
- 自动识别符合特病单议条件的病例（器官移植、罕见病、新技术、多学科治疗等）
- 按实际合规费用折算分值和结算金额
- 生成特病单议病例清单和申报材料模板
- 分析特病单议病例占比和费用情况

### 场景7：用户要“评估DIP3.0规则适配”
- 按DIP3.0先期分组、联合手术、特殊病组等新规则重新核算病例
- 对比新旧规则的入组率、分值变化、盈亏情况
- 输出规则适配评估报告和优化建议

## 回答时要主动补充的限制说明

- 国家规范给的是方法框架，地方目录、分值、点值、辅助目录细节以本地医保实际版本为准
- 本项目当前主要实现了主目录入组与点值核算，辅助目录调校和专家评议扣减如未实现，要明确标注“未纳入当前自动计算”
- 当点值缺失、目录版本不一致、编码版本混用时，核算结论只能作为试算或诊断，不应表述为正式结算结果

## 常见输出模板

### 核算结论

- 病例总数
- 完整命中数
- 部分命中数
- 未入组数
- 入组率
- 总分值
- 点值
- 预计结算金额
- 实际总费用
- 结余/亏损

### 问题分析

- 未入组的前 3 类原因
- 亏损最重的前 5 个科室或病组
- 高价值修复动作

### 管理建议

- 编码标准化
- 主手术补录
- 目录库版本同步
- 点值维护
- 对 partial / unmatched 病例建立人工复核清单

## 特殊类型病例判断标准
### 新生儿病例
- 判断条件：入院年龄 < 28天
- 需校验字段：年龄/出生日期、出生体重
- 处理逻辑：优先匹配新生儿目录，按出生体重分层校正
### 重症病例
- 判断条件：满足以下任意一条
  1. ICU住院天数 > 7天
  2. 有创呼吸机使用时长 > 96小时
  3. 存在CRRT（血液净化）治疗记录
- 需校验字段：ICU天数、呼吸机使用时长、手术/操作编码（CRRT）
- 处理逻辑：优先匹配重症病种，CCI系数按≥3级校正
### 肿瘤病例
- 判断条件：主诊断/副诊断为肿瘤类疾病（ICD编码C00-D48）
- 需校验字段：诊断编码、肿瘤分期、治疗方式（手术/放化疗/靶向等）
- 处理逻辑：匹配对应肿瘤病种，转移/晚期肿瘤应用校正系数
### 联合手术病例
- 判断条件：同一住院期间存在≥2个手术操作编码，且均为主要治疗相关手术
- 需校验字段：手术编码列表、手术级别
- 处理逻辑：优先匹配联合手术专属病种，无对应病种时按主手术入组
### 异常病例判断标准
#### 超高费用病例：住院总费用 > 所属病组均费 × 3
#### 超低费用病例：住院总费用 < 所属病组均费 × 0.3
#### 超长住院病例：住院天数 > 所属病组平均住院日 × 2 或 住院天数 > 60天
#### 二次入院病例：同一患者14天内因相同主诊断再次入院
#### 特病单议病例：满足以下任意一条
1. 器官移植、造血干细胞移植、人工肝、重度烧伤
2. 罕见病病例
3. 使用国家医保谈判药品/新耗材/新技术
4. 多学科联合治疗、院内转科、多部位手术
5. 住院天数超过60天的康复类病例
6. 其他符合本地医保特病单议政策的病例

## 禁止事项

- 不要把国临版编码直接当医保版编码用于正式入组
- 不要在点值缺失时硬算结算金额
- 不要把 `partial` 当成完全命中
- 不要脱离本项目库表口径随意改写字段名和结果定义
